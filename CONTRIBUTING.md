# Contributing to AnimioUI Component

Thanks for your interest in contributing.
We welcome bug reports, improvements, and new component ideas.

## How to Contribute

1. Fork the repository.
2. Create a new branch:

```bash
git checkout -b feat/your-feature-name
```

3. Make your changes and keep them focused.
4. Run checks locally:

```bash
npm run lint
npm run build
```

5. Commit with a clear message.
6. Open a Pull Request with:
   - What changed
   - Why it changed
   - Screenshots or GIFs (for UI updates)

## Contribution Guidelines

- Follow existing project structure and naming patterns.
- Keep components reusable and easy to read.
- Prefer small, reviewable pull requests.
- Do not commit secrets, tokens, or `.env` files.
- Update docs when behavior changes.

## Reporting Issues

When opening an issue, include:

- Clear title and description
- Steps to reproduce
- Expected vs actual behavior
- Environment (OS, browser, Node version)
- Screenshots where relevant

## Questions

If you are unsure where to start, open an issue and ask for guidance.
